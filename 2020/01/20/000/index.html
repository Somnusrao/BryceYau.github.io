记一次踩坑的日子

Posted on 2020-01-20

使用github博客功能踩坑日记

2020之前已经搭建过博客一次，但中途断了很长时间想重新优化。代码有做历史保留，但鉴于已经忘了整个搭建过程，想要重新梳理。在这过程中，突然发现踩了几次大坑，需要好好的记录下，方便后期规避。
使用hexo+github的搭建文章有很多，这里引用下我二次搭建的某个博主的文章

https://mogeko.me/2017/002/

下面是搭建过程的踩坑复盘：
1）hexo d 无法上传问题
A:该问题主要造成的原因是针对hexo init创造的blog目录下，_config.xml文件的repo字段值不正常导致

2）gitpages 404
该部分是地址解析不正确导致，除非你单独购买域名做解析，否则建议使用github.username作为你的仓库名

以下是调用mogeko的文章，仅为了方便其他人浏览作用。—特此申明

准备工作

  1. 一个 Github 账号。 <-我们要使用它提供的 Github Pages 服务来搭建博客 https://github.com/
  2. Node.js <- Hexo 依赖于此 https://nodejs.org/zh-cn/
  3. Git <- 用来发布博客 https://git-scm.com/
  4. Hexo <- 博客框架 https://hexo.io/zh-cn/index.html

安装 NexT 主题

使用 Hexo 的最大好处就是非常的自由，你可以完全掌握你的博客，想把它弄成什么样都可以。
因此，我们可以安装跟好看的主题来替换 Hexo 自带的 landscape 主题。
我推荐使用 NexT 主题（就是你们现在看到的样子），因为它简洁大方，比较符合我的审美。
而且它也是所有主题中，获得 stars 最多的。（在 Github 中，stars 越多，说明该项目越受欢迎。）

编写文章

引用一段来自官方的文档

你可以执行下列命令来创建一篇新文章。
$ hexo new [layout] <title>
您可以在命令中指定文章的布局（layout），默认为 post，可以通过修改 _config.yml 中的 default_layout 参数来指定默认布局。

创建文章后，可以看见 source\_posts\ 下有一篇新建的以 .md 为后缀文章

$ hexo d -g # 等同于 hexo deploy + hexo generate
